# FANZA作品発見サイト MVP（試作版）

## 今できること
- iPhone対応の検索画面
- キーワード、ジャンル、メーカー、並び順による絞り込み
- 「新作」「セール」「高評価」「独占」タグ検索
- お気に入り保存（ブラウザ内）
- 年齢確認画面
- FANZA誘導ボタンの配置

現在は画面確認用のダミーデータです。露骨な作品名・画像は入れていません。

## iPhoneで確認する方法
1. ZIPを解凍します。
2. `index.html` をSafariで直接開く場合、iPhoneの環境によっては制限があります。
3. 一番確実なのは、後でGitHub Pages、Cloudflare Pages、Netlifyなどへ公開する方法です。
4. 公開作業は画面を見ながら手順を案内できます。

## 実データ接続時に必要なもの
- DMMアフィリエイトID
- DMM WebサービスのAPI ID
- サイト名
- 公開先

API IDやアフィリエイトIDは、この試作版のJavaScriptへ直接書きません。
公開サイトに秘密情報を埋め込むと第三者に見えるため、実装時はサーバー側の環境変数へ保存します。

## 次の制作工程
1. サイト名・ロゴ・基本デザイン確定
2. サーバー側APIの作成
3. DMM Webサービスとの接続
4. 作品データをデータベースへ保存
5. 作品詳細、出演者、メーカー、ジャンルページを生成
6. アフィリエイトリンクとクリック計測
7. 管理画面
8. 公開、動作確認、DMMアフィリエイトのサイト登録

## 推奨構成
- フロント：Next.js
- データベース：Supabase PostgreSQL
- 公開：Vercel
- 定期取得：Vercel Cron または外部Cron
- 画像：DMM APIが返す正規画像URLを利用
- 秘密情報：環境変数で管理

## 注意
DMM Webサービスは1回の取得件数や一定時間内のリクエスト回数に制限があります。
毎回ブラウザから直接APIへ問い合わせる方式ではなく、サーバーが定期取得して保存したデータを表示する構成にします。
